             <h3 class="panel-title">About</h3>
           </div>
           <div class="panel-body">

`Sireum <http://sireum.org>`__ Logika is an
`open source <https://github.com/sireum/v3/tree/master/logika>`__
pedagogical tool being developed at
`SAnToS Laboratory <http://santoslab.org>`__,
`Kansas State University <http://k-state.edu>`_
to aid in teaching an introductory logics course in computer science (e.g.,
`CIS 301: Logical Foundations of Programming <http://proglogics.santoslab.org>`__).


The tool is accompanied by:
(1) a set of :ref:`course notes <course-notes>`,
which is an adaptation of the |ProgrammingLogics| lecture notes written by |Schmidt|, and
(2) an Integrated Verification Environment (IVE) for Logika -- :ref:`LIVE <live>`, built on top of
`JetBrains <https://www.jetbrains.com>`__' `IntelliJ IDEA <https://www.jetbrains.com/idea/>`__.
Logika uses the `Z3 <https://github.com/Z3Prover/z3>`__ theorem prover to
automatically solve algebraic formulae and to offer some level of automation suitable
for an introductory logics course.

Logika is inspired by:

* The `Natural Deduction Proof Checker (NDPC) <http://people.cis.ksu.edu/~schmidt/301s14/NDPC/ndpc-pe.jar>`__
  tool (see its
  `manual <http://people.cis.ksu.edu/~schmidt/301s14/NDPC/user_manual.pdf>`__)
  for propositional and predicate logics developed by Brian Mulanda,
  `Rodney Howell <http://people.cis.ksu.edu/~rhowell/>`__, and
  James Thompson.

* The `Floyd-Hoare Programming Logic Proof Checker <http://people.cis.ksu.edu/~schmidt/301s14/cis301.zip>`__
  developed by Schmidt that accompanies his |ProgrammingLogics| lecture notes.

Logika is a combination of both tools; it uses the same input language for
reasoning about propositional, predicate, and programming logics (i.e., the propositional logic
language is a subset of the predicate logic language, which in turn, is a subset of the
programming logic language).
The programming logic language itself is a subset of the `Scala <http://scala-lang.org>`__
programming language, whose programs can be run by using the regular Scala interpreter
equipped with the
`Logika runtime library <https://github.com/sireum/v3/blob/master/logika/jvm/src/main/scala/org/sireum/logika/package.scala>`__.
